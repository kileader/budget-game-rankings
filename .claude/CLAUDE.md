# Claude Project Handoff Rules

Use the repo handoff files as shared project memory.

Before non-trivial work, read:

1. `docs/HANDOFF.md`
2. `docs/DECISIONS.md`
3. `docs/NEXT_STEPS.md`

After meaningful work, update `docs/HANDOFF.md`.

When you make a durable decision, append it to `docs/DECISIONS.md`.

When priorities or the queue change, update `docs/NEXT_STEPS.md`.

Keep entries brief and practical.
